# Swarming

```
1. Problema: Multitasking, los miembro realizan tareas a la vez.
2. Un PB y un SB poco saludable: al final del Sprint tenemos demasiado "en proceso"
3. Con cada Sprint, acumulamos más tareas "En proceso" y aparentemente, vamos más lento.
4. Cada abeja tiene un rol que desempeñar en la colmena, pero trabajan como una sola entidad cuando es necesario para atacar una amenaza.
5. El patrón de enjambre es que se suman fuerzas para atacar una necesidad dedicando la mayor cantidad de miembros posibles.
6. Es útil para reducir el WIP
```

# Procedimiento de emergencia

```
1. Problema: No llegaremos al SG
-Posiblemente es debido a problemas emergentes, problemas técnicos, pérdida de personal, errores de estimación o priorización.
2. Ninguna problema es un problema, equivocate pronto y barato, y corrige pronto y barato.
3. Se debe notificar lo antes posible los problemas.
4. Corregir el error rápido, es corregir el error barato.
5. Que las personas no teman en equivocarse.

Procediiento de emergencia:
-Cambia la forma de desempeñarnos como equipo, la forma de planificar y ordenarnos cómo equipo.
-Delega una parte del trabajo a otro equipo que vaya muy adelantado.
-Reduce el alcance (-PBI de baja prioridad)
-Cance el Sprint y replanifica.
-Notificar cambios de fecha de entrega.
```
