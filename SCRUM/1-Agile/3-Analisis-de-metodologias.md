# Procesos

```
1. Trabajar agilmente se puede llevar a todos los enternos, pero a veces pues ser más conveniente optar por metologias tradicionales.
2. Los proyectos se caracterizan por tener un comeinzo, un final, un objetivo y unos recursos limitados.
3. Para llevar a cabo el proyecto se necesitan procesos de una secuencia definida de pasos con resultados medibles conocidos para poder multiplicar esos resltudas.
4. Hay que diferencia cuando el tipo de proyecto es de consumo único o continuo, por ejemplo no es lo mismo construir una casa que la jardinería.
```

# Piramide inicial

```
1.              Producto
                /     \
           Proyecto   Proyecto
             /           \
        Procesos         Procesos
           /               \
       Tareas              Tareas

2. En ocasiones el proyecto es más importante que el producto.
3. Dividir el proyecto por áreas.
4. Los proyectos estáran siendo creados en simultaneo con el producto.
5. Por ejemplo al constuir una casa, traer al albañil, al electricista, al que pone el gas, el plomero, etc al mismo tiempo. No abría problemas de construcción por ser un trabajo simultaneo, ahorrando tiempos y costos desde la misma producción.
```

# Proyectos simples vs complejos

```
1. Primero dividir el proyecto por la complejidad de este.
2. Los proyectos simples cumplen con estas caracteristicas:
    -Claridad del objetivo: Alta
    -Riesgos: Bajos
    -Innovacion: Baja
    -Exp y con: Alta
    -% Exito: Alta
Por ej: crear una casa
-Aqui es mejor utilizar algunas metodologias tradicionales tipo waterfall

3. Los proyectos complejos cumplen con estas caracteristicas:
    -Claridad del objetivo: Aceptable
    -Riesgos: Medios
    -Innovacion: Alta
    -Exp y con: Variada
    -% Exito: Variables
Por ej: El arte, un cuadro.
-Aqui es mejor utilizar algunas metodologias agiles
```

# Ciclo de deming

```
1. Plan Do Check Act plan Do Chek Act ...
2. Plan: El momento de planificacion y estimaciones se planifica lo neceserio para empezar a trabar y luego un poquito más. corto y mediano plazo. ¿Que necesito para empezar a trabajar?
3. Do: Empezar lo antes posible, para antes enfrentar los retos. Debe de haber alguien que nos pueda dar respuestas a los problemas que vayan surgiendo.
4. Check: Comprobar, momentos de reflexión y comprobar que hicimos bien y que se hizo mal. No ser insultante agresivo o insultante. Proceso de mejora continua. Sacar reuniones para sacar autocritica.
5. Act: Actuar reflexionemos sobre lo aprendido, ahora tomar las decisiones para mejarar lo sucedido.
```

# Triangulo de hierro

```
1.         tiempo
            /  \
           /    \
          /      \
         /        \
        /  Calidad \
       /            \
      /              \
   Alcance -------- Coste

2. El centro de cada proceso de produccion en un proyecto es la calidad del producto entrega.
3. El tiempo son los ciclos de trabajo que se emplean para poder ejecutar el proyecto.
4. El alcance es hasta que punto podes resolver la necesidad del cliente.
5. El coste, recursos generales y personas que intervienen para producir el proyecto.
6. Los procesos agile atacan el tiempo y el coste.
7. El cliente al dar un tiempo determinado y el precio que decide pagar, determinará el alcance. Si da poco tiempo y poca plata, tendrá poco alcance.
```

# Cono de la incertidumbre

```
1. Mientras más lejos estemos de la conclusión del proyecto, menos claridad habrá para planificar o prometer.
2. No asegure estimaciones a los comienzos. OJOOo
3. Sigue este flujo:
 -Definición inicial: con mucha incertidumbre, pero poco coste.
 -Definición inicial aprobada: Un poco menos de incertidumbre, pero el comienzo supone un aumento en el coste
 -Primeros requisitos: Se acerca a un producto completado, menos incertidumbre, pero comienza un coste significativo.
 -Desarrollo inicial: Cada vez menos incertidumbre, más cerca del producto completado y un coste ya muy establecido.
 -Desarrollo avanzado: El mayor punto del coste, más cercano a comlpetar el proyecto y una incertidumbre casi nula. En este punto es cuando ya puede empezar a dar promesas, cuando puede terminar.
4. Este cono puede ocurrir al contrario. Mayores costos al principio y menos al final. Vea la inversión de Disney al adquirir Fox.
5. Lo ideal es que el coste se pueda mantener.
```

# Enfoques Predictivos

```
1. Trabajan en etapas consecutivas y diferenciales, con poca o nula intervención del cliente.
2. Se le pide a la empresa un producto, la empresa se pierde y vuelve con el producto realizado. El cliente no es un actor principal.
3. Por ejemplo cuando se hizó la mona lisa. 
```

# Enfoques Adaptativos

```
1.Se adpta con la perspectiva del cliente.
2. Se divide en ciclos ya sean iterativos o incrementales
3. Itearativo: Se trabaja con breves entregas con una creación GENERAL de todo el producto y se va mejorando. Se entregan prototipos que pueden fallar.
4. Incremental: Se trabaja con breves entregas con una creación muy detallada de una parte del proyecto y luego se pasa a otra parte hasta terminar el proyecto. Es necesario tener muy presente cual es producto al que se quiere llegar.
5. Iterativo + incremental = SCRUM :D
```
